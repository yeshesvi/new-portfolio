Alternative workflow for Phone number verification

Interaction Design

The problem

As verification and personalization is making its way into everyday apps and services, users are expected to provide and verify their email and phone number.

I have noticed the following is a common way to do this verification,

  • Input Phone number
  • Wait for a code
  • Switch over to the messenger apps
  • Remember / copy and input the code

Total number of interactions (taps) required: 5 or 7

The Alternative

The following is one of the alternatives, that worked well for me during user testing.

  • Input Phone number
  • Hit the “verify” button
  • App places a call to our server and ends it immediately ( no cost to the user )
  • User receives confirmation and its done

Total number of interactions: 2

Problems with the alternative workflow

The only issue is when the user is on a tablet device when registering, as most tablets are not capable of placing phone calls. Other than that, there are no realistic issues with this workflow. It is highly efficient and keeps the user engaged with in the app.
